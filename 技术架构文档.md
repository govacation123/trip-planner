# 智能旅行助手 — 技术架构文档

基于 **LangGraph** + **LangChain** 构建的智能旅行规划助手，集成高德地图 REST API，支持多轮对话优化、场景化长期记忆与群体智慧推荐。

    │                              │

    │              │                               │

**节点职责**：

| 节点 | 职责 | 调用工具 |
|------|------|---------|
| `weather_agent` | 查询目的地城市天气预报 | `amap_search_tool(type="weather")` |
| `hotel_agent` | 搜索推荐酒店 5-8 家 | `amap_search_tool(type="poi")` |

## 三、三层记忆架构

### 3.1 设计原则

> **硬约束 > 软约束，永远不让记忆覆盖用户指令**

| 层级 | 存储介质 | 生命周期 | 作用 |
|------|---------|---------|------|
| **工作记忆（Working Memory）** | LangGraph State | 单次对话 | 保障当前行程数据一致性 |
| **短期记忆（Short-term）** | `state.history: List[str]` | 滑动窗口 | 解决多轮对话指代消解 |
| **长期记忆（Long-term）** | SQLite (`user_memory` 表) | 30 天 TTL | 场景化画像持久化 |

### 3.2 长期记忆数据结构

```sql
-- SQLite: user_memory 表
user_id   TEXT PRIMARY KEY  -- 即 session_id（前端 localStorage 生成）
profile   TEXT              -- JSON 字符串
updated_at TIMESTAMP         -- 最后活跃时间，用于 TTL 过期判断
```

`profile` JSON 结构（按场景隔离）：

```json
{
  "scenarios": {
    "亲子度假": {
      "住宿": "民宿",
      "饮食禁忌": "无辣",
      "节奏偏好": "宽松"
    },
    "商务出差": {
      "住宿": "五星级",
      "预算": "1000+"
    }
  }
}
```

### 3.3 记忆读取引擎

```python
# 精准召回：仅获取指定场景的记忆
def get_scenario_memory(user_id: str, scenario: str) -> dict

# 群体智慧：聚合全网同类场景 Top-K 偏好
def get_popular_scenario_preferences(scenario: str, top_k: int = 3) -> dict
```

### 3.4 记忆写回策略

```python
# Refine 时由 LLM 推断场景，增量合并到该场景下
save_scenario_memory(user_id, scenario, extracted_preferences)
```

## 四、约束优先级体系

Planner 和 Refiner 遵循同一套优先级规则，发生冲突时严格按序服从：

```
优先级 1 [硬约束-Highest]    用户当前输入的指令
优先级 2 [硬约束-WorkMem]    当前计划中的硬性参数（城市/天数/基础预算）
优先级 3 [软约束-Soft]      场景化召回的长期记忆（relevant_memory）
优先级 4 [辅助-Optional]     群体智慧 Top 偏好（crowd_memory）
```

## 五、MCP 轻量工具注册体系

### 5.1 架构

```
LLM (bind_tools)
    │
    ▼
amap_search_tool  ──►  TOOLS_REGISTRY["amap_search_tool"]
                              │
                              ▼
                        execute_tool(name, args)
                              │
                              ▼
                        AmapService (高德 REST API)
```

### 5.2 工具定义

```python
@tool
def amap_search_tool(query: str, city: str, search_type: str) -> str:
    """
    高德地图综合搜索工具。

    - search_type="weather"  → amap_service.get_weather(city)
    - search_type="poi"      → amap_service.search_poi(query, city)
    """
    ...
```

### 5.3 注册与调用

```python
TOOLS_REGISTRY: dict[str, Any] = {}   # 全局注册表

def register_tool(name: str, tool_func: Any) -> None:
    TOOLS_REGISTRY[name] = tool_func

def execute_tool(tool_name: str, args: dict) -> str:
    tool_func = TOOLS_REGISTRY.get(tool_name)
    result = tool_func.invoke(args)
    return str(result)
```

### 5.4 Agent 提示词（MCP 工具调用）

| Agent | 提示词关键指令 |
|-------|--------------|
| `weather_agent` | "请调用系统提供的高德地图(amap)工具来查询天气" |
| `hotel_agent` | "请调用系统提供的高德地图(amap)工具搜索真实的酒店数据" |
| `attraction_agent` | "请务必调用系统提供的高德地图(amap)相关工具来获取真实数据" |

## 十一、注意事项

1. **TTL 过期机制**：用户 30 天无活跃操作（Refine），SQLite 中的记忆自动清理
2. **群体智慧依赖数据量**：初期用户少时 `get_popular_scenario_preferences` 返回空字典，属正常现象
3. **最小修改原则**：Refiner 只修改用户提到的部分，不重写整个计划
4. **场景推断**：由 LLM 通过 `with_structured_output(MemoryExtraction)` 自动判断，无需用户手动选择
